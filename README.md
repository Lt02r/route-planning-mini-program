# 旅游路线规划微信小程序

## 功能介绍

### 🗺️ 地图标点
- 在地图上显示所有保存的标点
- 长按地图任意位置快速添加标点
- 点击标点查看详情（名称、分类、优先级、备注、坐标）
- 支持从标点详情直接编辑或删除

### 🏷️ 标点分类显示
- 支持6种分类：景点、餐饮、住宿、购物、交通、其他
- 地图页顶部可按分类过滤显示标点
- 不同分类使用不同颜色的图钉标记

### 📋 标点增删改查
- **添加**：点击地图页右下角「＋」按钮，或长按地图，或在标点管理页添加
- **查看**：在标点管理页浏览所有标点，支持搜索和排序
- **编辑**：在标点详情或标点列表中点击进入编辑页
- **删除**：在地图详情弹窗或标点列表中删除

### 📝 自定义备注与优先级
- 每个标点支持自定义备注（最多200字）
- 支持3个优先级：普通（灰）/ 重要（橙）/ 必去（红）
- 在所有列表视图中均可看到优先级标识

### 🚗 路线规划
- 在「路线」页勾选想要前往的标点
- 支持按选择顺序规划路线
- **智能优化**：使用最近邻算法自动优化路线顺序，减少总路程
- 显示路线地图、总距离、预计出行时间
- 支持驾车/步行两种出行方式

### 🤖 AI推荐路线
- 根据可用游玩时间（1-24小时）和每个景点平均游玩时间自动计算可游玩数量
- 按优先级自动筛选最值得去的标点
- 支持按分类偏好过滤候选标点
- 自动生成优化路线，以时间轴形式展示
- 可切换到路线规划页查看完整路线

## 技术实现

- **框架**：微信小程序原生框架
- **数据存储**：wx.setStorageSync / wx.getStorageSync 本地持久化
- **路线优化**：最近邻贪心算法（Nearest Neighbor Heuristic）
- **距离计算**：Haversine 公式（球面三角法）
- **地图**：微信原生 map 组件，支持标注、折线、点击交互

## 🚀 部署与测试指南

### 一、前置要求

| 工具 | 说明 | 下载地址 |
|------|------|----------|
| 微信开发者工具 | 必须，用于模拟器预览和上传发布 | [下载](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html) |
| 微信小程序 AppID | 必须，注册后获取 | [注册](https://mp.weixin.qq.com/wxopen/waregister?action=step1) |
| Node.js ≥ 16 | 可选，仅运行单元测试时需要 | [下载](https://nodejs.org/) |

---

### 二、配置 AppID（必须步骤）

> ⚠️ **project.config.json 中的 AppID 为占位符，必须替换后才能运行。**

1. 前往 [微信公众平台](https://mp.weixin.qq.com/) 注册或登录小程序账号
2. 进入「开发管理」→「开发设置」，复制 **AppID**
3. 编辑项目根目录下的 `project.config.json`，将：
   ```json
   "appid": "your_appid_here"
   ```
   替换为：
   ```json
   "appid": "wx你的真实AppID"
   ```

---

### 三、在模拟器中运行（本地开发测试）

1. 打开**微信开发者工具**，选择「小程序」→「导入项目」
2. **项目目录**：选择本仓库根目录（包含 `app.js` 的目录）
3. **AppID**：填入你的真实 AppID（或选择「测试号」体验部分功能）
4. 点击「确定」，项目将自动编译并在模拟器中显示

**模拟器测试要点：**

| 功能 | 操作步骤 |
|------|----------|
| 添加标点 | 长按地图任意位置 → 填写标点信息 → 保存 |
| 浏览标点 | 切换到「标点」tab → 搜索/排序/过滤分类 |
| 路线规划 | 切换到「路线」tab → 勾选标点 → 点击「智能优化路线」|
| AI推荐 | 切换到「AI推荐」tab → 设置可用时间 → 生成推荐路线 |
| 获取位置 | 地图页点击定位按钮（模拟器中会提示授权） |

**注意：** 模拟器中 `wx.chooseLocation()`（选择位置）默认返回北京坐标，真机测试可获取真实位置。

---

### 四、在真机上预览（真机调试）

1. 在微信开发者工具顶部工具栏，点击「预览」
2. 用**注册该小程序的微信账号**扫描弹出的二维码
3. 即可在真实手机上运行，支持 GPS 定位等真实设备功能

### 五、单元测试（Utils 层）

项目对核心工具函数（`utils/route.js` 和 `utils/storage.js`）提供了基于 **Jest** 的单元测试，可在无需微信开发者工具的情况下验证算法正确性。

#### 安装依赖

```bash
npm install
```

#### 运行测试

```bash
# 运行所有测试
npm test

# 运行测试并生成覆盖率报告
npm run test:coverage
```

```

Test Suites: 2 passed, 2 total
Tests:       56 passed, 56 total
```

### 六、发布上线（生产部署）

1. 完成功能测试后，在微信开发者工具中点击「上传」
2. 填写版本号和版本描述
3. 登录 [微信公众平台](https://mp.weixin.qq.com/) → 「版本管理」→ 提交审核
4. 微信官方审核通过（通常 1–7 个工作日）后，点击「发布」即可上线

---

### 七、已知限制

- **数据仅存本地**：所有标点数据保存在设备本地存储（上限约 10 MB），不支持跨设备同步
- **位置权限**：`获取位置` 和 `选择位置` 功能需要用户授权，拒绝后相关按钮不可用
- **路线算法**：使用最近邻贪心算法，对于大量标点（>50个）结果可能非最优
- **时间估算**：驾车按 40 km/h、步行按 5 km/h 固定速度估算，不考虑实时路况
